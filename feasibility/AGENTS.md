# feasibility/AGENTS.md

本目录维护课题可行性验证材料。进入本目录前先读根目录 `AGENTS.md`，再读本文件和 `README.md`。

## 作用

- 验证当前课题方向是否有可观察的系统信号。
- 验证本地环境、依赖、数据库连接、脚本 dry-run 和 smoke run 是否可用。
- 保留 Ray / Arrow / object transfer / fan-in 等组件级 microbenchmark。
- 为 `motivation/` 的端到端主动机实验提供早期证据、负结果和计时边界校准。

## 边界

- 这里是“可行性验证”，不是论文最终有效性验证。
- 这里不放 GPU-backed E2E 主动机结果。
- 这里不放正式系统画像、瓶颈定位或优化收益结论。
- 这里的结果只能说明方向可继续探索、组件有局部信号或环境可用，不能直接外推为真实 GPU-backed 数据库 AI 算子链路瓶颈。

## 子目录

- `benchmarks/`：组件级可行性脚本和公共工具。
- `results/`：可行性结果、连接验证、smoke CSV 和自动报告。

## 规则

- 新可行性实验必须有明确问题、运行命令、参数说明、CSV 输出和结论边界。
- warm-up 要忽略或明确标注。
- 连接验证只能证明系统可用，不能写成性能收益。
- 如果结果会影响课题主线判断，应迁移或登记到 `motivation/results/`，并按动机结果规则重写报告。
