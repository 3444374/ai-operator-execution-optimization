# overview/AGENTS.md

本目录维护项目总纲、当前路线和阶段计划。进入本目录前先读根目录 `AGENTS.md`。

## 作用

- 记录课题定位、用户目标、阶段计划和方向边界。
- 记录当前证据链的高层结论。
- 不放原始 CSV，不放长篇文献摘录，不放代码实现细节。

## 更新规则

当以下内容变化时，同步更新 `current_direction_and_plan.md`：

- 课题方向或候选技术路线变化；
- 业务场景变化；
- 重要实验结论变化；
- PG18.4 / PG18.3 链路状态变化；
- baseline 优先级变化。

`project_outline.md` 保持稳定，只记录高层定位和目录结构。

## 当前边界

- 主线服务 AI infra / inference infra 能力积累。
- 数据库场景是落地入口，不把项目带回传统数据库内核 GPU 查询算子。
- 当前推荐主线是：基于 Ray/Daft/Lance 类外部执行链路的数据库 AI 算子批处理系统调优。
- Ray/Daft/Lance 是候选系统和调优对象，不是已经被证明必须采用的最终产品路线。
- PG18.4 已完成连接验证和第一组 4096 行系统画像；连接验证与系统画像必须分开引用。

## 证据分级

每个判断要区分：

- 本地实验事实；
- 模拟实验事实；
- 合理推断；
- 待确认问题。

不得把 PG18.4 本地结果写成 PostgreSQL 18.3 内部平台结果。
