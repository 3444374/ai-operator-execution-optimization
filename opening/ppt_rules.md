# 开题 PPT 制作规则

## 模板使用

- 学校模板是版式约束，不是背景图片。
- 优先复用母版、占位符、主题字体、页眉页脚。
- 不要把模板页截图后铺满当背景。
- 不要重复绘制模板里已有的校徽、横线、页脚和装饰块。
- 先确定安全内容区，再放正文、图表和流程图。

## 页面结构

推荐按问题驱动组织：

```text
为什么这个题值得做
  -> 现有研究做到哪一步
  -> 还缺什么
  -> 本课题准备怎么做
  -> 当前准备到什么程度
  -> 后续计划和风险控制
```

每一部分回答一个问题：

| 部分 | 回答的问题 |
|---|---|
| 背景 | 为什么需要研究 |
| 现状 | 别人做了什么 |
| 不足 | 还有什么没解决 |
| 方案 | 我怎么切入 |
| 可行性 | 当前准备到什么程度 |
| 计划 | 后面怎么完成 |

## 页面内容

- 一页只突出一个重点。
- 表格页必须有结论句。
- 图示页必须有核心路径。
- 页面文字用关键词和短句，详细解释放备注。
- 如果一页超过 6 到 8 行正文，优先拆页或改成图表。
- 表格只保留汇报需要的列，不追求全。
- 边界和不能夸大的结论要主动写，必要时放页面底部或答辩备注。

## 图与图片

- 解释流程时画流程图。
- 需要比较时用矩阵表。
- 展示阶段关系时用路线图。
- 展示风险时用“风险-控制方式”表。
- 图形优先用 SVG 或 PPT 原生形状，避免低清截图。
- 图中每个节点只放关键词，解释放讲稿备注。
- 图表必须包含图表头、颜色含义、横纵坐标或表头。

## 颜色与风格

- 主色跟随学校模板。
- 强调色只用于关键结论、关键路径或风险点。
- 一页内不要出现太多颜色层级。
- 学术开题 PPT 的目标是清楚，不是海报效果。

## 备注规则

每页备注默认包含：

```text
汇报讲稿：
这一页现场怎么讲，解释页面之间的因果关系。

答辩备注：
老师可能问什么，应该怎么答，哪些边界必须主动说明。
```

讲稿不要照抄页面文字。页面负责“看”，备注负责“讲”。

## 最终检查

- 模板元素没有被重复绘制。
- 正文没有压住页眉、页脚、校徽。
- 所有页面标题位置一致。
- 图表和正文都在安全内容区内。
- 每页只有一个主结论。
- 文字不过密，表格不过宽。
- 关键页有汇报讲稿和答辩备注。
- PPTX 已用 WPS/PowerPoint 实际打开检查。
- PPTX 与报告正文、飞书版、本地 Markdown 口径一致。
