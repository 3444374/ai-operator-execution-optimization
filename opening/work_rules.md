# 开题工作规则

## 工作目标

开题阶段要证明：

- 题目有研究价值。
- 问题定义清楚。
- 技术路线可执行。
- 前期准备有基础。
- 后续计划能落地。
- 风险和边界自己知道。

开题不是最终成果发布。更稳的说法是“当前实验作为可行性依据，后续会扩大验证”，不要说“已经完整证明”。

## 并行任务线

| 任务线 | 目标 | 主文件 |
|---|---|---|
| 开题报告 | 满足学校模板，完整说明研究背景、现状、目标、内容、方案、进度和预期成果 | `report/opening_report.md` |
| 开题 PPT | 现场讲清楚题目、问题、路线、可行性和计划 | `slides/opening_ppt.md` |
| 飞书进度汇报 | 面向日常同步，说明最近进展、问题和下一步 | `feishu/progress_update.md` |
| 精读文献 | 支撑背景、现状和答辩抽问 | `literature/reading_list.md` |
| 答辩问答 | 准备高频问题和边界口径 | `qa_bank.md` |
| 项目日志 | 记录非实验类材料修改 | `logs/project_log.md` |

## 写作原则

- 少用空泛判断。不要只写“具有重要意义”，要写清楚重要在哪里、对应什么问题、后续怎么验证。
- 研究内容要可执行。每一项研究内容最好能对应一个方法模块、评价方式或阶段任务。
- 创新点要克制。开题阶段写“预期创新点”，不要写成已经完成的创新成果。
- 所有结论都要有边界：当前能说明什么、不能说明什么、后续怎么验证。
- 报告、PPT、飞书版、Word 版不能出现不同题目、不同研究内容或不同实验口径。

## 高频答辩策略

老师质疑时，回答结构优先采用：

```text
先承认问题存在
  -> 说明当前为什么这样处理
  -> 说明后续如何补充验证
```

不要和老师争“已经证明了”。开题阶段主动承认边界通常更稳。

## 最终检查

- 题目、研究目标、研究内容一致。
- 报告结构符合学校模板。
- 参考文献数量、外文比例和精读文献数量满足要求。
- 15 篇精读文献能讲清楚。
- PPT 和报告口径一致。
- 技术路线能对应后续实验和实现任务。
- 创新点没有夸大。
- 局限和风险已主动写入。
- 答辩备注准备了常见问题。
- 所有非实验类修改已记录到 `logs/project_log.md`。
